You have successfully connected IcM to your agent ({1}) in PCD. If this incident was not routed to the correct IcM team, you will need to create a Routing Rule to ensure future incidents are routed appropriately. Routing can be applied by specifying either ownerId, agentId, or assetGroupId in the routing rule. At least one is required.

To do so, you will need to be a Tenant Admin or Rule Manager in IcM for your service, or find your Service Admin.

Create a Routing Rule

  1. Navigate to IcM -> Administration -> Routing Rules
  2. Select Add a Routing Rule
    • Service: [your service]
    • Cloud Instance: (All)
    • Routing Id: ownerId:{5}
    • Correlation Id: (blank)
    • Role: [provide assetGroupId listed in incident]
    • Environment: (All)
    • DC/Region: (All)
    • Slice: (blank)
    • Instance/Cluster: agentId:{6}
    • Enforce Default Severity: (unchecked)
    • Escalate to Service: [your service]
    • Escalate to Team: [your team]

Mapping Table*

  • Routing Id = ownerId:[guid]
  • Role = assetGroupId:[guid]
  • Instance/Cluster = agentId:[guid]

*ownerId will always be provided. The assetGroupId and agentId will only be provided if it applies.

For more information on Routing Rules, please refer to IcM wiki.
